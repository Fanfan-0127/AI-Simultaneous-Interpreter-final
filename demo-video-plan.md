# AI 同声传译助手 — Demo 视频规划

## 1. 视频目标

- **时长**: 2-3 分钟（竞赛演示最佳时长）
- **受众**: 竞赛评委（技术背景，关注创新性、完成度、用户体验）
- **核心信息**: 极低延迟 + 多语言 + 精美字幕 = 专业级同声传译桌面工具

## 2. 视频结构（5 幕）

### 第一幕：问题引入（15 秒）

**画面**: 一段英文技术演讲/会议的播放画面，屏幕下方没有任何字幕

**旁白/字幕**:
> "参加国际技术会议、观看英文演讲时，语言障碍让信息获取效率大打折扣。
> 现有的翻译工具延迟高、体验差，无法满足实时需求。"

**转场**: 画面缩小，AI 同声传译助手主窗口弹出

---

### 第二幕：核心演示 — 英→中实时翻译（45 秒）⭐ 最重要

**画面布局**: 上半部分播放 YouTube 英文技术演讲 / 播客，下半部分展示悬浮字幕窗

**操作流程**:
1. 主窗口：选择音频源（WASAPI Loopback 扬声器）
2. 点击"开始"按钮 → 状态栏变绿 ● 正在监听
3. 演讲音频播放中 → 悬浮窗实时弹出双语字幕

**关键亮点捕捉**:
- **低延迟**: 演讲者话音刚落（~0.5 秒），中文字幕已出现 → 给延迟特写镜头
- **草稿→修正**: 同一句先出草稿翻译，句子完整后自动修正（画面切换至主窗口"修正列表"，展示 ASR 修正 + MT 修正两列）
- **音频电平**: 状态栏音频指示器实时跳动，证明在实时采集

**拍摄手法**:
- 屏幕录制 + 画中画（右下角放演讲源画面）
- 悬浮窗字幕用慢动作回放展示延迟

---

### 第三幕：多语言切换演示（30 秒）

**操作流程**:
1. 打开"偏好设置" → 展示三个 Tab（API / 音频与翻译 / 显示）
2. 切换识别语言：英文 → 日文（下拉框展示 28 种语言）
3. 切换翻译目标：中文 → 英文
4. 保存后回到主界面，播放日语音频 → 悬浮窗显示日文原文 + 英文翻译

**关键亮点**:
- 28 种 ASR 源语言 + 92 种 MT 目标语言
- 切换后即时生效，无需重启

**画面**: 分屏对比——左侧日→英，右侧上一步的英→中

---

### 第四幕：视觉定制能力（25 秒）

**操作流程**:
1. 打开"偏好设置" → "显示"Tab
2. 逐一展示可配置项：
   - 原文/译文颜色选择器
   - 字号调节（原文 26pt → 译文 28pt）
   - 字体选择
   - 背景不透明度（滑块拖到半透明 → 可以看到下方桌面内容）
   - 暗色/亮色主题切换
3. 实时预览：每次修改保存后悬浮窗即时更新

**关键亮点**:
- 毛玻璃半透明悬浮窗
- 拖拽移动 + 锁定位置
- 鼠标穿透（锁定时可点击悬浮窗下方窗口）
- 新句子从底部淡入动画（`easeOutCubic` 200ms）

**拍摄手法**:
- 拖动悬浮窗到不同位置
- 展示"锁定悬浮窗"后鼠标穿透效果

---

### 第五幕：专业功能 + 收尾（25 秒）

**画面**: 主窗口全貌

**快速展示**:
1. **字幕记录表格**: 三列（时间 | 原文 | 译文）滚动展示整场会话
2. **修正专区**: 展开"修正列表"，展示 ASR 识别修正 + MT 翻译修正的完整追踪
3. **术语提取**: 点击"查看术语" → 弹出高频术语列表
4. **导出字幕**: "保存字幕" → 生成 `subtitles-20260607-143052.txt`
5. **延迟显示**: 状态栏 `延迟: ASR 120ms | MT 180ms | Total 300ms`

**结尾画面**:
- 悬浮窗显示最后一句翻译
- 主窗口 + 悬浮窗 + 导出文件 三者同框
- 淡出，出现文字：

> **AI 同声传译助手**
> 28 语言识别 · 92 语言翻译 · <500ms 延迟
> Java 21 · DashScope Qwen · 开源

## 3. 拍摄清单

### 硬件准备

| 项目 | 用途 |
|------|------|
| Windows 11 电脑 | 运行演示 |

| 项目 | 说明 |
|------|------|
| OBS Studio | 屏幕录制，支持多场景切换 |

### 演示素材建议

| 场景 | 推荐素材 | 原因 |
|------|---------|------|
| 英→中 | TED: "The Future of AI" 类演讲 | 技术词汇多，展示翻译质量 |
| 日→英 | NHK 新闻片段 | 句子短、语速适中，翻译效果好 |
| 技术术语 | Google I/O / Apple WWDC 回放 | 展示术语一致性 |

```powershell
# 系统音频设置

# 3. 将系统音量调到合适的水平
```

## 5. 录制技巧

### OBS 场景设置

```
场景 1: "问题引入"      — 全屏浏览器 + 无字幕会议画面
场景 2: "核心演示"      — 上半屏视频 + 下半屏悬浮窗 + 右侧主窗口
场景 3: "多语言"        — 设置对话框特写 + 分屏翻译效果
场景 4: "视觉定制"      — 设置对话框"显示"Tab + 悬浮窗实时变化
场景 5: "专业功能"      — 主窗口全貌 + 字幕表格 + 导出
场景 6: "结尾"          — Logo + 文字
```

### 画面要点

- **光标高亮**: OBS 启用鼠标点击高亮效果，让观众看清操作
- **关键区域放大**: 延迟数字、修正列表等关键信息用 OBS 滤镜放大
- **平滑转场**: 场景切换使用 300ms 淡入淡出
- **帧率**: 60fps 录制，确保动画流畅
- **分辨率**: 1920×1080（1080p）

### 避免的问题

- ❌ 不要让系统通知弹窗打断演示 → 开启"专注助手"
- ❌ 不要展示 API Key → 用环境变量，设置对话框中 Key 已脱敏
- ❌ 不要让翻译结果空白太久 → 提前测试网络和 API
- ❌ 不要让悬浮窗遮挡关键画面 → 调整位置或临时拖走
- ❌ 不要用太快的演讲素材 → 选语速适中的 TED Talk
- ❌ 不要忘记展示"修正"过程 → 这是技术亮点

## 6. 备选方案（如果现场出问题）

| 风险 | 应对 |
|------|------|
| API 超时/网络波动 | 提前录制一段翻译结果，用本地回放模拟实时 |
| WASAPI 设备不可用 | 备选 Java Sound 音频源 |
| 悬浮窗位置不对 | 提前设置好窗口位置并锁定 |
| 日文素材播放问题 | 准备英→中单语言演示作为兜底方案 |
| 系统卡顿 | 关闭不必要后台程序，OBS 用硬件编码 |
